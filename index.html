01 — About

동작하는 것과 신뢰할 수 있게 동작하는 것

LLM 기반 추천 서비스와 추천 시스템 실험을 진행하며, 기능이 동작하는 것과 신뢰할 수 있게 동작하는 것은 다르다는 점을 경험했습니다. 모델 응답 형식, 추천 서비스 호출, 데이터 저장소, 캐시, Docker 실행 환경, CI 검증이 하나의 흐름으로 이어질 때 서비스는 더 안정적으로 동작합니다.

모델을 실험하는 데서 멈추지 않고, 왜 동작하는지·어디서 실패할 수 있는지· 다시 검증할 수 있는지를 설명할 수 있는 백엔드·AI 개발자로 성장하고 있습니다.

TailorPlay

LLM 기반 Steam 게임 추천 서비스에서 백엔드와 실행 환경을 연결.

문제

LLM Agent의 호출 조건·응답 형식이 흔들리고, 로컬에선 되던 기능이 Docker·CI 환경에선 다르게 실패했습니다. 모델·추천 서비스·데이터·실행 환경을 하나의 안정적 흐름으로 묶어야 했습니다.

나의 역할(팀)

백엔드와 실행 환경 연결 담당 — LLM Agent와 추천 서비스 사이 응답 경계 정리, 컨테이너·CI 구성.

트러블슈팅

Steam API·추천 서빙·저장소 경로를 역할별로 분리해 요청 흐름을 단순화. 로컬↔CI 차이로 인한 실패는 Docker Compose로 구성요소를 함께 실행하고 GitHub Actions 검증을 연결해 원인을 좁혔습니다.

결과 & 배운 점

반복 요청·세션 상태는 Redis 캐시로 보조하되, AI 기능의 안정성은 모델 호출이 아니라 데이터·캐시·실행 환경의 경계에서 결정된다는 점을 배웠습니다.

Book Rating Prediction RMSE 3위

도서 평점 예측 competition에서 텍스트·이미지 피처를 결합.

문제

도서 평점 예측에서 제목·요약 같은 텍스트와 표지 이미지 등 이질적인 피처를 어떻게 결합하고, 실험을 재현 가능하게 비교할지가 과제였습니다.

나의 역할(팀)

텍스트·이미지 피처 결합과 실험 관리 흐름 담당.

트러블슈팅

SBERT로 메타데이터 의미 정보를 임베딩하고 표지 이미지는 CNN/ResNet 분기로 별도 처리해 결합. Stratified K-Fold로 분할을 관리하고 WandB로 설정·결과를 기록해 모델별 비교를 재현 가능하게 만들었습니다.

결과 & 배운 점

RMSE 기준 3위(네이버 커넥트재단 주관). 추천 문제를 한 번의 점수가 아니라 반복 검증의 대상으로 다루는 법을 익혔습니다.

PublicPlus

Spring Boot 기반 공공시설 검색·활동 참여·커뮤니티 백엔드 API.

문제

공공시설 검색·활동 모집/참여·커뮤니티를 하나의 백엔드로 묶되, 인증·실시간·외부 연동을 서비스 기능에 맞게 연결해야 했습니다.

나의 역할(팀)

백엔드 API 구현 — 조회·인증·외부 연동 흐름 구성.

트러블슈팅

JPA·QueryDSL로 조건 기반 조회와 도메인 접근을 정리하고, Naver Search 같은 외부 검색은 서비스 계층에서 분리해 관리. Security/JWT/OAuth2로 인증을, Redis로 상태성 데이터를 보조하고 WebSocket/STOMP·Calendar·Firebase로 실시간/알림 기능을 연결했습니다.

결과 & 배운 점

외부 연동과 실시간 기능을 서비스 계층 단위로 분리할수록 변경·확장이 쉬워진다는 점을 체감했습니다.
